# 算法讲解清单与指南

> 面向**小鱼（FisherGoGo）**的算法教学规范。  
> 每次前辈请求讲解一个算法知识点时，严格按此清单执行。  
> 核心约束：**纯文字环境，无法画图**。所有可视化必须用文字替代。

---

## 一、前置原则

1. **先痛后悟**：不先报算法名字，先用题目场景制造"旧工具失效"的张力。
2. **一句话核心**：任何算法必须能浓缩成一句人话，在正式展开前先给出来。
3. **推导优先**：关键公式/结论让前辈自己推，我只给骨架和引导。
4. **即学即用**：讲完后立刻推 1-2 道可以直接开写的题目。
5. **前辈视角**：始终以前辈的认知习惯（结构敏感、代码手、重视识别力）设计讲解路径。

---

## 二、讲解五步流程

### 第 1 步：场景触发（不报算法名）

**目的**：让前辈先尝试用自己的知识库解决问题，感受"旧工具失效"的瞬间。

**模板**：

> 来看一道题：
>
> 「[用一句中文描述题目场景，去掉所有专业术语]」
>
> 约束：$n \le$ [范围]，$H \le$ [范围]。
>
> 暴力怎么做？复杂度多少？为什么不可行？
> 有没有更优做法？

**要求**：
- 题目必须是经典模板题或变式题，来源注明（如 P2371、ABCxxxF）
- 数据范围必须给出，让前辈自己判断复杂度是否可行
- 等前辈回应后再进入第 2 步（除非前辈明确说"直接讲"）

---

### 第 2 步：一句话核心 + 直觉建立

**目的**：用最少的文字建立算法的核心直觉，让前辈"嗯，合理"。

**模板**：

> 这个算法的核心思想用一句话概括：
>
> 「[一句话，不超过 40 字]」
>
> 具体来说：[2-3 句话展开核心洞察，不涉及实现细节]

**要求**：
- 一句话必须是中文，不用公式
- 展开部分可以用一个具体的小例子说明（数字演示）
- 这一步**不给代码、不给公式推导、不给建图细节**

**纯文字替代图示的方法（选用）**：

- **环形图替代**：用编号列表 + 箭头模拟
  ```
  节点 0 ──(+3)──> 节点 3 ──(+3)──> 节点 1
    ↑                                    │
    └────────────(+3)────────────────────┘
  ```
- **层次图替代**：用缩进表示层级
  ```
  root
    ├── left
    │   ├── left-left
    │   └── left-right
    └── right
  ```
- **流程替代**：用编号步骤 + 条件分支
  ```
  1. 取 min{a_i} 作为模数 m
  2. 初始化 dis[0]=0, 其余 = INF
  3. 对每个 x: 枚举 a_j, 松弛 (x+a_j) % m
  4. 统计: floor((H - dis[x]) / m) + 1
  ```

---

### 第 3 步：形式化推导（建图 / 建模）

**目的**：把直觉翻译成严谨的数学模型，让前辈理解"为什么对"。

**模板**：

> **建模关键**：
> - 状态定义：[每个节点/状态代表什么]
> - 转移：[从状态 A 到状态 B 的代价/条件]
> - 初始条件：[起点是什么]
> - 目标：[最终要求什么]
>
> **复杂度分析**：
> - 状态数：$O($...$)$
> - 转移数：$O($...$)$
> - 总复杂度：$O($...$)$

**要求**：
- 使用 LaTeX 数学公式
- 关键推导步骤编号（1. 2. 3.），让前辈可以逐条消化
- 如果涉及模运算、取整等容易出错的点，**显式标注并解释**
- 给出正确性的一句话证明（核心不变量）

---

### 第 4 步：代码骨架（关键细节留白）

**目的**：给可运行的代码框架，但故意留 1-2 个关键处让前辈自己填。

**模板**：

> ```cpp
> // [算法名] — 模板
> // 适用： [一句话场景]
> // 复杂度：O([...])
> 
> [包含关键变量声明和核心逻辑]
> // TODO: 前辈请完成 [某一步]
> ```
>
> **留白处**：[说明前辈需要自己完成的逻辑]
>
> **常见坑点**：
> - [边界条件 1]
> - [初始化陷阱]
> - [溢出/取模问题]

**要求**：
- 代码使用 C++（前辈的主力语言）
- 代码风格与前辈的 ACM 模板保持一致
- 留白处用 `// TODO: 前辈请...` 标记
- 给出至少 2 个常见坑点
- 如果算法有多版本（如线段树数组版 vs 指针版），默认给最通用的版本

---

### 第 5 步：题目推送 + 变式识别指南

**目的**：让前辈立刻验证学习效果，并建立"识别该算法"的能力。

**模板**：

> **立刻去写**：
> - 模板题：[题号 + 题目名 + 链接]（难度：⭐）
> - 进阶题：[题号 + 题目名 + 链接]（难度：⭐⭐）
>
> **识别指南**：看到以下关键词时考虑此算法：
> - [触发词 1]：[为什么]
> - [触发词 2]：[为什么]
> - [数据范围特征]
>
> **常见伪装**：
> - "[题面故事]"类问题，本质上就是 [核心模型]

**要求**：
- 模板题必须是 1 次 AC 就能过的（不卡常、不出偏门边界）
- 进阶题需要稍作转化才能套用
- 触发词至少 3 个
- 给出至少 1 个"题面完全不同但本质相同"的伪装案例

## 三、纯文字可视化工具箱

| 需求 | 替代方案 |
|------|----------|
| 图结构 | ASCII 箭头图 / 缩进树形图 |
| 流程 | 编号步骤 + 条件分支 |
| 数学推导 | LaTeX 公式 + 逐行注释 |
| 动画/动态过程 | 分帧文字描述 + "第 k 轮"标注 |
| 对比表 | Markdown 表格 |
| 空间结构 | 网格字符图（如 DP 表格） |

**ASCII 图快速参考**：
```
箭头: ──>  ──>  ──>
竖线: │
转角: └──  ├──
环形:  ┌──┐  节点0 → 节点3 → 节点1
      │  │         ↑              │
      └──┘         └──────────────┘
```

## 四、执行检查清单

每次讲完算法后，逐条自查：

- [ ] 第 1 步：是否先给了题目场景而非算法名字？
- [ ] 第 2 步：是否有一句话核心，且不超过 40 字？
- [ ] 第 3 步：状态/转移/复杂度是否全部明确？
- [ ] 第 4 步：是否有留白让前辈自己完成？
- [ ] 第 5 步：是否推了至少 1 道模板题 + 1 道进阶题？
- [ ] 纯文字约束：是否避免了"如图所示""看这张图"等无效描述？
- [ ] 坑点：是否标注了至少 2 个常见错误？

## 五、特殊场景处理

### 前辈说"直接讲，跳过题目"

→ 省略第 1 步，从第 2 步开始。但结束时仍推题目。

### 算法本身就是图论/几何类（天然需要图）

→ 第 2 步必须给出 ASCII 图 + 文字描述的双重表达。宁可多写几行字符画，不能让前辈靠想象。

### 算法需要多轮迭代展示（如 DP 填表）

→ 用分帧文字描述：
```
初始: dp = [0, INF, INF, INF, INF]
第1轮(考虑物品1): dp = [0, 3, INF, INF, INF]
第2轮(考虑物品2): dp = [0, 3, 5, 8, INF]
...
```

### 前辈追问细节

→ 自由展开，但仍保持结构化（用编号、分点），不陷入散乱聊天。
