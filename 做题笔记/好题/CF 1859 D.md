# 题解：Capygrad Portals 逃亡问题

## 一、题目模型抽象

给定一条数轴和 $n$ 个传送门。每个传送门可以用四个参数 $[l_i, r_i, a_i, b_i]$ 表示，且满足 $l_i \le a_i \le b_i \le r_i$。

**运作机制：** 如果你当前身处区间 $[l_i, r_i]$ 内的任意一点，你可以通过传送门跃迁到区间 $[a_i, b_i]$ 内的**任意一点**。传送门可以使用无数次。

**核心目标：** 给定 $q$ 个起点 $x$，求从每个起点出发，能够到达的最右侧（坐标最大）的位置。

## 二、核心推导与贪心性质

本题的核心在于发现跳跃方向的**单调性**和传送门之间的**依赖关系**。

### 1. 贪心性质：目的地最大化

既然我们的目标是向右逃得越远越好，那么当我们使用第 $i$ 个传送门时，我们应该跳到 $[a_i, b_i]$ 中的哪个点呢？

直觉上我们会选择跳到 $b_i$。但有没有可能跳到 $[a_i, b_i]$ 中的某个中间点 $y$（$y < b_i$），反而能接触到某个上限更高的传送门 $j$ 呢？

**结论是：不可能。**

**证明：** 假设存在一个传送门 $j$，能从 $y$ 跳到更远的 $b_j$（即 $b_j > b_i$）。既然传送门 $j$ 能在 $y$ 处被触发，说明 $l_j \le y \le r_j$。

同时，题目保证了 $b_j \le r_j$。既然 $b_j > b_i$，那么必定有 $r_j > b_i$。

因此，区间 $[l_j, r_j]$ 的左边界 $l_j \le y < b_i$，右边界 $r_j > b_i$，这意味着 $b_i$ **必定也完全包含在** $[l_j, r_j]$ **的范围内**！

所以，任何从 $y$ 能够使用的更优传送门，从 $b_i$ 也一定能使用。**我们只需要关心从整个区间** $[a_i, b_i]$ **出发能到达的最远距离即可。**

### 2. 状态转移：逆序处理消除后效性

如何求出“进入某个传送门后，最终能到达的最远距离”？

如果传送门 $i$ 最终能带我们到达极远点，它必然是直接跳到了 $b_i$，或者借由某个 $b_j > b_i$ 的传送门 $j$ 继续向右跳。

这构成了一个典型的 DAG（有向无环图）转移。为了消除后效性，我们只需要**将所有传送门按照** $b_i$ **从大到小排序**。

当我们处理第 $i$ 个传送门时，所有比它“更靠右”的传送门都已经处理完毕了。此时，区间 $[a_i, b_i]$ 里记录的最大可达坐标，就是后续跳跃的全局最优解。

## 三、算法设计与数据结构

根据上述推导，我们需要维护一个支持以下两种操作的数据结构：

1. **区间查询：** 询问当前区间 $[a_i, b_i]$ 内，能够到达的最远端点坐标（即 `max(b_i, query(a_i, b_i))`）。
    
2. **区间修改（取 Max）：** 既然传送门 $i$ 能带我们达到上述最远坐标，那么对于能进入传送门 $i$ 的入口范围 $[l_i, r_i]$，它们的最远可达坐标都可以更新为这个最大值（即 `update(l_i, r_i, val)`）。
    

这显然是**线段树（区间修改，区间求最大值）**的拿手好戏。

由于坐标值域高达 $10^9$，但 $n$ 和 $q$ 只有 $2 \times 10^5$，节点分布稀疏。为了避免动态开点线段树带来的巨大常数和空间开销（极易 MLE 或 RE），我们需要使用**离散化**。

## 四、代码实现详解

本代码的实现逻辑非常优雅，关键步骤如下：

1. **全量离散化：**
    
    将所有的 $l, r, a, b$ 以及查询的 $x$ 全部丢进一个 `vals` 数组中进行排序去重。
    
    ```c++
    sort(vals.begin(), vals.end());
    vals.erase(unique(vals.begin(), vals.end()), vals.end());
    ```
    
    巧妙之处在于，线段树的**区间下标使用的是离散化后的 ID**（$1 \sim m$），但线段树节点中**维护的 `lazy` 和 `d` 存储的依然是真实的坐标值**。这就省去了查询后再将 ID 映射回真实值的麻烦。
    
2. **线段树建树与初始化：**
    
    使用 `vector` 动态分配离散化后所需大小的 $4$ 倍空间。由于全部是求最大值，初始值设为 0 即可。
    
    ```c++
    Seg S(m); 
    int rt = 1; // 根节点从 1 开始，避免 ls=x*2 为 0 导致死循环
    ```
    
3. **逆序 DP 转移（核心逻辑）：**
    
    将所有线段按 $b_i$ 升序排序，然后使用 `df(i, v.size()-1, 0)` 倒序遍历（即按 $b_i$ 降序处理）：
    
    ```c++
    // val 为: 自身的终点 b_i 与 区间 [a_i, b_i] 内能跳到的最远距离 取最大值
    ll val = max(v[i].b, S.qy(rt, 1, m, a, b));
    // 将这个最远距离更新给入口区间 [l_i, r_i]
    S.up(rt, 1, m, l, r, val);
    ```
    
4. **回答询问：**
    
    经过所有传送门的区间覆盖后，数轴上每个点的最大可达距离已经固定。对于每个起点 $x$，只需查询它本身在线段树中的单点值，并与 $x$ 自身取 max（如果不进任何传送门，最远就是原地）。
    
    ```c++
    cout << max(Q[i], S.qy(rt, 1, m, x, x)) << " ";
    ```
    

## 五、复杂度分析

- **时间复杂度：** * 排序与离散化：整合了 $4n + q$ 个坐标，排序复杂度为 $O((N+Q) \log (N+Q))$。
    
    - 线段树操作：共 $N$ 次区间修改和区间查询，$Q$ 次单点查询，每次操作 $O(\log (N+Q))$。
        
    - 总时间复杂度：$O((N+Q) \log (N+Q))$，对于 $2 \times 10^5$ 的数据量，可以在 1 秒内轻松通过。
        
- **空间复杂度：** 线段树使用了大小为 $4 \times (4n+q)$ 的 `vector` 数组。总空间复杂度为 $O(N+Q)$，极致省空间，完全杜绝了动态开点带来的空间不可控问题。
    

## 完整代码参考

```c++
#include<bits/stdc++.h>
#define f(i,a,b) for(int i=a;i<=b;i++)
#define df(i,a,b) for(int i=a;i>=b;i--)
#define cmax(a,b) a=max(a,b)
#define cmin(a,b) a=min(a,b)
#define lowbit(x) ((x)&(-(x)))
#define cntbit(x) __builtin_popcount(x)
using namespace std;
typedef long long ll;
const int N=2e5+10;
const ll M=998244353;
int TT=1;

ll qp(ll a,ll x){
    ll res=1;for(;x;x>>=1,a=a*a%M)
        (x&1)&&(res=a*res%M);return res;
}

struct Line{
    ll l,r,a,b;
};

int n,q;

struct Seg{
    #define mid ((l+r)/2)
    #define ls x*2
    #define rs x*2+1
    int node=0;
    vector<ll> lazy,d;
    
    Seg (int _n){
        lazy.resize((_n+1)<<2,0);
        d.resize((_n+1)<<2,0);
    }
    
    void push_up(int x){
        d[x]=max(d[ls],d[rs]);
    }
    void push_down(int x){
        cmax(d[ls],lazy[x]);
        cmax(lazy[ls],lazy[x]);

        cmax(d[rs],lazy[x]);
        cmax(lazy[rs],lazy[x]);
    
        lazy[x]=0;
    }
    void up(int x,ll l,ll r,ll ql,ll qr,ll val){
        if(ql<=l&&r<=qr){
            cmax(d[x],val);
            cmax(lazy[x],val);
            return ;
        }
        push_down(x);
        if(ql<=mid) up(ls,l,mid,ql,qr,val);
        if(qr>mid) up(rs,mid+1,r,ql,qr,val);
        push_up(x);
    }
    ll qy(int x,ll l,ll r,ll ql,ll qr){
        if(ql<=l&&r<=qr){
            return d[x];
        }
        push_down(x);
        ll ans=0;
        if(ql<=mid) cmax(ans,qy(ls,l,mid,ql,qr));
        if(qr>mid) cmax(ans,qy(rs,mid+1,r,ql,qr));
        return ans;
    }

    #undef mid
};

const ll inf=1e9;

void solve(){
    cin>>n;
    vector<Line> v;
    
    vector<ll> vals;
    f(i,1,n){
        ll l,r,a,b;
        cin>>l>>r>>a>>b;
        v.push_back({l,r,a,b});
        vals.push_back(l);
        vals.push_back(r);
        vals.push_back(a);
        vals.push_back(b);
    }
    cin>>q;
    vector<ll> Q(q+1,0);
    f(i,1,q) {
        cin>>Q[i];
        vals.push_back(Q[i]);
    }
    sort(vals.begin(), vals.end());
    vals.erase(unique(vals.begin(), vals.end()), vals.end());
    auto get_id = [&](ll &x) {
        x=lower_bound(vals.begin(), vals.end(), x) - vals.begin() + 1;
    };

    int m=vals.size();
    Seg S(m);

    sort(v.begin(),v.end(),[&](const Line& x,const Line& y){
        if(x.b==y.b) return x.a<y.a;
        return x.b<y.b;
    });
    
    int rt=1;

    df(i,v.size()-1,0){
        auto [l,r,a,b]=v[i];
        get_id(l);get_id(r);get_id(a);get_id(b);
        ll val=max(v[i].b,S.qy(rt,1,m,a,b));
        S.up(rt,1,m,l,r,val);
    }

    f(i,1,q){
        ll x=Q[i];
        get_id(x);
        cout<<max(Q[i],S.qy(rt,1,m,x,x))<<" ";
    }
    cout<<"\n";
}

int main(){
    std::ios::sync_with_stdio(0);
    std::cin.tie(0);
    cin>>TT;
    while(TT--) solve();
    return 0;
}
```
