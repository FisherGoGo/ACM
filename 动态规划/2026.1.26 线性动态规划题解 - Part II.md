# K

- 我们现设 $P_{i}= \frac{p_{i}}{q_{i}}$ 表示每条线段出现的概率，$Q_{i}=1-P_{i}$ 表示第 $i$ 条线段不出现的概率
- 初始概率是全不选 $No=\prod Q_{i}$ ，那么当线段 $i$ 由不出现变为出现，就要乘上 $\dfrac{P_{i}}{Q_{i}}$ ，我们令 $k_{i}=\dfrac{P_{i}}{Q_{i}}$ 
- **状态设计：** $dp[i]$ 表示覆盖完前 $i$ 个单元格，且覆盖方式满足“每个格点恰好被覆盖一次”的概率贡献总和
- **状态转移：** 为了覆盖到位置 $r$，必须存在某条线段以 $r$ 结尾，且其左端点 $l$ 之前的位置 $[1, l-1]$ 已经被完美覆盖
	- 这里 `dp[a[i].l-1]` 代表了前 $l-1$ 个位置被完美覆盖的各种组合的概率权值和
	- 乘上 `a[i].k1` 代表接上当前这条线段
$$dp[a[i].r] = (dp[a[i].r] + dp[a[i].l-1] \times k_{i}) \pmod{mod}$$
- **边界与结果：**
	- **初始化：** `dp[0] = No`。即在没选任何线段前，基础概率是所有线段都不发生的概率。
	- **排序：** `sort(a+1, a+n+1, cmpr)` 按右端点 $r$ 升序排列。这是线性 DP 的常规操作，确保计算 $dp[r]$ 时，所有可能的 $dp[l-1]$ 已经计算完成。
	- **最终输出：** `dp[m]`，即恰好完美覆盖到最后一个格子的概率总和
# L

- **状态设计：** $dp[i]$ 表示以第 $i$ 个元素为结尾，且保留第 $i$ 个元素不改的情况下，修改前 $i-1$ 个元素使其满足非降序要求的**最小总成本**
- **状态转移：** 为了计算 $dp[i]$，我们需要枚举上一个**被保留**的元素位置 $j$：
	- **约束条件：** $a_j \le a_i$（保证非降序）
	- **中间成本：** 位于 $j$ 和 $i$ 之间的所有元素（$j+1$ 到 $i-1$）都必须被修改，其总代价为 $pre[i-1] - pre[j]$
	- **转移方程：** 
$$dp[i] = \min_{0 \le j < i, \ a_j \le a_i} \{ dp[j] + (pre[i-1] - pre[j]) \}$$

> [!NOTE] 
> - 问题转化为：寻找一个原数组的子序列，满足非降序条件，且该子序列中元素的 **$c_i$ 之和最大**
> - $i$ 的范围是 $1$ 到 $n+1$（其中 $n+1$ 是为了方便结算设置的虚拟哨兵位）
> - **辅助结构：**
> 	- **前缀和 `pre[i]`：** 预处理 $c_i$ 的前缀和，以便 $O(1)$ 时间计算区间修改总代价。  
> 	- **哨兵位 `a[n+1] = INF`：** 强制在最后增加一个无穷大的数，这样 $dp[n+1]$ 就能自动汇总保留任意合法子序列后的最小修改成本
# M

- **状态设计：** $dp[i][j]$ 表示处理到从高到低第 $i$ 位时，当前的进位状态为 $j$ 的期望步数
	- $i$：当前处理的二进制位数（从 $1$ 到 $n$）
	- $j \in \{0, 1\}$：$j=1$ 表示该位当前值为 1，$j=0$ 表示该位当前值为 0
- **状态转移逻辑：** 每一步操作都会减少一位，所以步数首先要 $+1$。我们要考虑的是当前位 $s[i]$ 在接收到低位的进位影响后，变成 0 或 1 的期望
	1. **若当前位 $s[i] == '1'$：**
	    - **维持 $1$ 状态：** 只有低位**没有**通过操作 2 产生进位，这一位才保持 1
	        - 方程：`dp[i][1] = (dp[i-1][1] + 1)`
	    - **变为 $0$ 状态（发生进位）：** 如果低位进位了，则这一位 $1+1$ 变成 $0$ 并继续向更高位进位。进位发生的概率是基于前一步两种状态的平均
	        - 方程：`dp[i][0] = ((dp[i-1][1] + dp[i-1][0]) * inv2 + 1)`
	2. **若当前位 $s[i] == '0'$：**
	    - **维持 $0$ 状态：** 低位没进位，依然是 0。
	        - 方程：`dp[i][0] = (dp[i-1][0] + 1)`。
	    - **变为 $1$ 状态：** 低位进位了，0 变成 1。
	        - 方程：`dp[i][1] = ((dp[i-1][1] + dp[i-1][0]) * inv2 + 1)`
- **边界条件：** `dp[1][1] = 1`，`dp[1][0] = 0`。因为最高位一定是 1（题目保证），且处理到最高位变为 1 时任务结束
# N

- 操作一次的增量是
$$
cntbit(old)-cntbit(2^l)+cntbit(new)
$$
- 若操作 $k$ 次，那么中间许多项可以消除，只剩下
$$
cntbit(old)-cntbit(final)+k
$$
- 若 $k$ 够多我们完全可以把其加到 $2$ 的整数次幂形式，我们考虑 $k$ 很小的情况，这个情况大概是 $k\le 64$ 
- 我试过贪心但是不行，那么考虑 DP
- 我们设 DP 状态 
$$
dp[i][j][c]
$$
- 表示在进行到第 $i$ 位，已经用了 $j$ 次操作，此时来自 $i-1$ 的进位有 $c$ 次时能减少的最多的 $1$ 
- 那么在考虑将状态转移时我们这样转移，我们要从 $i$ 转移到 $i+1$
	- 先看原始位 bit
	- 枚举已用次数 j 和上一位进位 c 和操作次数 t （在这一位加几个 1）
	- 那么这一位实际值 Bit 为 (bit+c+t)%2
	- 给下一位的进位为 (bit+c+t)/2
	- 这一位的贡献为 bit-Bit
	- $dp[i+1][j+t]\left[ \frac{bit+c+t}{2} \right]$ 与 $dp[i][j][c]+bit-Bit$ 取 $\max$ 
- 最后看最高位，枚举对最高位的进位 $c$ 
- 最大减少值为
$$
dp[64][k][c]-cntbit(c)
$$
- 为什么是 $cntbit(c)$ ，因为相当于加 $c\times 2^{64}$ ，高位肯定是全空的
# O

- 可见行操作只影响垂直相邻的建筑，列操作只影响水平相邻的建筑
- 分情况 DP 即可
# P

- 首先 $x$ 肯定是先变小变成一个 $z$ ，再变大成 $y$ 
- 这个 $z$ 最优是 $\gcd(x,y)$ 
- 那么我们令 $x \leftarrow \frac{x}{g}$ 和 $y \leftarrow \frac{y}{g}$ 
- 现在问题变成令 $x$ 变成 $1$ 再变成 $y$ ，因为乘法和除法在步数计算上是**对称**的，因此本质上都是在求解：**将一个数 $n$ 通过分解为若干个不大于 $k$ 的因子乘积，所需的最少因子个数** 
- 我们设 $dp[i]$ 表示变成达到 $x$ 的第 $i$ 个因数所需最小步数，显然我们从小往大推，若 $g[j]$ 整除 $g[i]$ 且商小于等于 $k$ ，那么可以转移
$$dp[i] = \min(dp[i], dp[j] + 1)$$
- 再对 $y$ 求一遍即可
# Q

- 由于约束只存在于相邻塔之间，我们可以定义：
	- $predp[i]$：在位置 $i$ 建立一座塔，且满足 $[0, i]$ 范围内所有约束的最小成本。
	- $sufdp[i]$：在位置 $i$ 建立一座塔，且满足 $[i, n+1]$ 范围内所有约束的最小成本。
### 1. 转移方程

- 对于 $predp[i]$，它必须由前一个塔 $j$ 转移而来，且满足 $i - j \le k$：
$$predp[i] = a_i + \min_{i-k \le j < i} \{predp[j]\}$$
- 同理，对于后缀：
$$sufdp[i] = a_i + \min_{i < j \le i+k} \{sufdp[j]\}$$
### 2. 滑动窗口优化（单调队列）

- 直接计算 DP 的复杂度是 $O(nk)$。由于 $n$ 可达 $5 \times 10^5$，我们需要优化。

- 观察转移方程，这是一个**区间最小值**查询。代码中使用了 `std::deque`（双端队列）来维护一个单调递增的队列，使得每次获取 $\min$ 的操作复杂度降为 $O(1)$。总预处理时间复杂度为 $O(n)$。

## 处理询问 $(p, v)$

当某个位置 $p$ 的成本改变时，受影响的不仅仅是 $p$ 点，还有可能跨过 $p$ 点的路径。代码将情况分为两部分处理：
### 情况 A：强制经过修改点 $p$

如果最小代价的路径必须经过 $p$（或者代码中 `s[p]=='1'` 的逻辑，虽然原题描述中没有强制点，但代码中预留了这种判断），那么总代价非常简单：

$$\text{Cost} = (predp[p] - a_p) + v + (sufdp[p] - a_p)$$

即：$p$ 左侧的最优解 + $p$ 的新成本 + $p$ 右侧的最优解。
### 情况 B：跨越点 $p$ 的局部重算

- 这是代码中最核心的部分。当 $a_p$ 改变，可能会影响 $p$ 附近的一段决策。
- 代码在询问中开启了一个局部循环，范围大概是 $[p-k, p+k]$。
	- 它尝试在 $[p-k, p+k]$ 这个小窗口内模拟一次小型 DP。
	- 使用单调队列 `dq` 维护从左侧 $predp$ 传过来的最小值。
	- 对于窗口内的每一个点 $i$，计算：`当前局部DP值 + sufdp[i]`，并更新全局最小值。
    
**注意**：代码中有一段 `if(s[p]=='1')` 的逻辑，这通常对应题目中某些点“必须选”的情况。如果题目没有要求必须选，通常需要考虑**不经过 $p$ 点直接跨过去**的可能性。
