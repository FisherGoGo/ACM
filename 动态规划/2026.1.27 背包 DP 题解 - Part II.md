# L

- 会发现如果我们有一组拿取物品的方案，那么我们免费拿的物品可能是按照代价从小到大排序过后的后 $k$ 个
- 依次我们先对原数组按照代价从小到大排序，那么对于任意方案数，其在这个数组中肯定是前半花钱拿后半免费拿，全部聚集在这个数组的同一边
- 那么我们对这个数组跑一个以代价为限制的前缀背包，再跑一个以数量为限制的后缀背包，枚举分界点即可
# M

- 典型的背包 DP
- 可见只会有一个道具被切割，既其贡献不是 $0$ 也不是 $w_{i,p_{i}}$ 
- 我们设 $dp[i][0/1]$ 其中 $dp[i][0]$ 表示已经装了 $i$ 的容量，没有物品被切割放入的最大增益，而 $dp[i][1]$ 表示已经装了 $i$ 的容量，有物品被切割放入的最大增益
- 那么答案就是 $\max\{\max\limits_{0\le i\le k} \{dp[i][0]\},dp[k][1]\}$
- 转移即可
# N

- 变种 0/1 背包问题
- 设 $dp[j]$ 表示当实习生 A 承担的任务总耗时为 $j$ 分钟时，实习生 B 承担的任务**最小**总耗时
- 我们先考虑把所有任务丢给实习生 B ，即 $dp[0]=\sum\limits b[i]$
- 那我们转移任务的时候实习生 B 的耗时会减少 $b[i]$ ，实习生 A 会增加 $a[i]$ ，那么转移方程
$$
dp[j]=\min\{dp[j],dp[j-a[i]]-b[i]\}
$$ 
- 算完后枚举找最优搭配即可
# O

- 设 $dp[i][j]$ 表示 $x$ 总共除了 $2^{i}$ 和 $y$ 总共除了 $2^{j}$ 所耗费的最小值，很普通的背包 DP
- 随后枚举最终除的数判断取最小即可
# P

- 一个多重集 $s$ 是合法的，当且仅当：**所有没有出现在 $s$ 中的颜色 $x$ 的总出现次数 $cnt_x$，都不能超过 $s$ 中所有元素在原集合 $a$ 中的总出现次数，** 即：$\max_{x \notin s} \{cnt_x\} \le \sum_{c \in s} cnt_c$ 
- 我们将上条件改为 $\sum\limits_{c\in s}cnt_{c}\ge \max\limits_{1\le i\le n}\{cnt_{i}\}$ ，因为若最大值在 $s$ 中，上式子自然成立，也满足上一条的式子，如果最大值不在 $s$ 中，那么我们只考虑 $\sum\limits_{c\in s}cnt_{c}\ge \max\limits_{1\le i\le n}\{cnt_{i}\}$ 的部分即可
- 我们设状态 $dp[i]$ 表示从原数组中每种数字**最多选一个**，组成一个集合 $S$，使得这些颜色在原数组中对应的**频率之和**恰好等于 $j$ 的方案数，那么对于枚举的 $i,j$ ，我们这样子转移即可
$$
dp[j]=dp[j]+cnt[i]*dp[j-cnt[i]]
$$
- 为什么我们要乘 $cnt[i]$ ，因为同一种数字虽然相同，但是他们出现的不同次数会做相应贡献，所以我们要乘上
- 最后答案为 $\sum\limits_{maxcnt}^{n} dp[i]$ 
# Q

- 对于一个点 $v$，我们可以将他的兄弟结点放在他之前，因为他的兄弟结点在他之前，所以他的兄弟结点的子树的所有结点都在他之前
- 我们考虑背包，$dp_{u,i,j,k}$ 为 $u$ 的子结点中不包含 $k$ 这个子结点，有 $i$ 这个结点在 $k$ 之前，并且结点的总数为 $j$ ，显然这个 dp 是 $O(n^{4})$
- 可以算出包含 $k$ 结点的 $dp_{u,i,j}$ 然后通过回退背包操作得到 $dp_{u,i,j,k}$ 这样复杂度便是 $O(n^{3})$ 然后 $dp_{u,i,j}$ 便是普通的背包 dp, 然后对于 $dp_{u,i,j,k}$，我们将其维护成一个 $DP_{u,k,v}$ 含义为从 $u$ 到 $k$ 结点个数为 $v$ 个的方案数 
- 我们维护一个 $f_{i,j}$ 指的是在 $i$ 点为第 $j$ 个的总方案数，不难发现 $f_{1,1}=\Pi^{1}_n(dep_{out}(i)!)$ 然后对于 $u \rightarrow v$ 我们去除自由的方案数，再加上我们计算出的方案数所以 dp 方程为 $f_{i,j+k+1}+=f_{i,j}+\frac{DP_{u,v,k}}{deg(u)!}$
# R

- 最后三维前缀和优化查询即可
