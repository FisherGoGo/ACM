## 第〇讲 · 前置知识回顾

- 基础线段树：建树、单点修改、区间查询、pushup
- **懒标记（Lazy Propagation）**：区间修改 + pushdown 机制
  - 核心流程：进入节点先 pushdown，离开前 pushup
  - 多标记问题注意 pushdown 先后顺序（先乘后加 vs 先加后乘）
- **标记永久化**：懒标记不下传，在查询路径上叠加标记信息
  - 适用条件：修改可快速叠加、与顺序无关（如区间取 max）
  - 优势：减少 pushdown 带来的新节点创建，常用于可持久化线段树
- 离散化基础（扫描线、主席树的前置技能）

> 📖 详见笔记 [线段树.md § 标记永久化](./线段树.md)

## 第一讲 · 动态开点线段树

- **核心动机**：值域巨大（如 $[1, 10^9]$）但操作次数有限时的空间优化
- **关键思想**：不预建整棵树，访问时才创建节点；用 `ls[x]` / `rs[x]` 替代固定索引 `2p` / `2p+1`
- **核心实现**：`update` 时若节点为空则 `++node` 动态创建
- **易漏点**：`pushdown` 时子节点也要动态创建，不可遗漏
- **复杂度**：时间 $O(q \log V)$，空间 $O(q \log V)$，一般开 `q * 40` 节点

## 第二讲 · 权值线段树

- **本质**：以数值为下标建立线段树，而非以数组位置为下标
- **核心操作**（全部可在 $O(\log V)$ 完成）：
  - `insert(x)`：值 $x$ 对应位置 +1
  - `kth(k)`：线段树二分，查找全局第 $k$ 小 → 利用 `d[ls[x]]` 与 `k` 比较决定走左走右
  - `rank(x)`：查询 $[1, x-1]$ 区间和
  - `predecessor(x)` / `successor(x)`：尽量往右/往左找
- **通常与动态开点结合使用**
- **经典例题**：用权值线段树替代平衡树（插入、删除、排名、第 K 大、前驱、后继）

> 📖 题解 `K` 有完整权值线段树实现平衡树的代码

## 第三讲 · 主席树（可持久化线段树）

- **核心思想**：每次单点修改只新建 $O(\log n)$ 个节点，其余节点复用旧版本
- **实现要点**：
  - `modify(pre, x, ...)` 从旧版本 `pre` 拷贝信息到新节点 `x`
  - 修改路径上创建新节点，未修改的子树直接指向旧节点
  - 每个版本保留独立的根 `root[ver]`
- **经典应用 — 静态区间第 K 小**：
  1. 离散化后按顺序插入权值线段树
  2. 前缀 `[1, i]` 对应版本 `root[i]`
  3. 利用版本可减性：`root[R]` − `root[L-1]` = 区间内值域分布
  4. 两棵线段树同时向下二分
- **复杂度**：时间 $O((n+q) \log n)$，空间 $O(n \log n)$
- **扩展 — 树上路径查询**：维护根到每个节点的前缀主席树 `s[u]`，路径 `(u,v)` 信息 = `s[u] + s[v] - s[lca] - s[fa[lca]]`

## 第四讲 · 线段树合并

- **场景**：将两棵动态开点权值线段树的内容合并为一棵
- **核心实现**：
  ```cpp
  int merge(int a, int b, int l, int r) {
      if (!a || !b) return a | b;
      if (l == r) { d[a] += d[b]; return a; }
      ls[a] = merge(ls[a], ls[b], l, mid);
      rs[a] = merge(rs[a], rs[b], mid + 1, r);
      pushup(a); return a;
  }
  ```
- **复杂度**：均摊 $O(n \log n)$，每次合并至少销毁一个节点
- **经典应用**：
  - **树上差分 + 合并**：雨天的尾巴 — 路径加袋子 → 四个点差分标记 → 自底向上合并
  - **并查集 + 合并**：动态维护连通块内第 K 小
  - **树上统计**：替代 dsu on tree

> 📖 题解 `E`、`F`、`G` 均为线段树合并实战

## 第五讲 · 线段树分治

- **核心思想**：将「时间轴」作为线段树下标，离线处理有时效性的操作
- **适用场景**：每条操作只在某个时间段 $[L, R]$ 内有效，需要在每个时刻回答查询
- **核心流程**：
  1. 将每个操作插入到线段树上覆盖其有效时间段的 $O(\log T)$ 个节点
  2. DFS 遍历线段树，进入节点时应用该节点的所有操作
  3. 到达叶节点 $(l=r=t)$ 时回答时刻 $t$ 的查询
  4. 回溯时撤销操作（用栈记录修改历史并回滚）
- **关键配合**：**可撤销并查集**（按秩合并，不路径压缩）
  - 每次 merge 将修改前的状态压入栈
  - 回溯时弹栈恢复 fa 和 height
- **复杂度**：每条操作被加入 $O(\log T)$ 次，整体 $O(q \log T \cdot \log n)$
- **经典应用**：
  - 动态图连通性（离线）：每条边有出现时间和消失时间
  - 二分图判定：扩展域并查集 + 线段树分治

## 第六讲 · 扫描线

- **核心思想**：将二维几何问题降维，沿 $x$ 轴扫描，用线段树维护 $y$ 轴的覆盖信息
- **事件模型**：每个矩形化为两条竖线
  - 左边界 $(x_1, y_1, y_2, +1)$：进入矩形，该 $y$ 区间覆盖次数 +1
  - 右边界 $(x_2, y_1, y_2, -1)$：离开矩形，该 $y$ 区间覆盖次数 −1
  - 事件按 $x$ 坐标排序后逐个处理
- **线段树维护内容**：
  - `cnt`：当前节点对应 $y$ 区间被完全覆盖的次数
  - `len`：当前节点对应 $y$ 区间内被覆盖的总长度
  - `pushup` 关键逻辑：若 `cnt > 0` 则 `len = 原始区间长度`，否则 `len = len[ls] + len[rs]`
- **为什么不需要 pushdown**：`cnt` 是整体覆盖计数，加减对称，天然适合标记永久化
- **关键步骤**：
  1. 离散化所有 $y$ 坐标（扫描线的线段树建立在离散化后的 $y$ 轴上）
  2. 相邻两个 $x$ 事件之间的贡献 = `当前覆盖长度 × (x[i+1] - x[i])`
- **经典应用**：
  - **矩形面积并**：$\sum \text{len} \times \Delta x$，扫一遍累加
  - **矩形周长并**：同时维护横向和纵向，横向 = $\sum |\text{len}_\text{new} - \text{len}_\text{old}|$，纵向需额外维护线段数量
  - **矩形面积交**：维护「被覆盖 ≥ 2 次」的长度
- **复杂度**：$O(n \log n)$（排序 $O(n \log n)$ + 线段树 $O(n \log n)$）
- **例题推荐**：洛谷 P5490（矩形面积并模板）、洛谷 P1856（矩形周长并）

## 第七讲 · 线段树上维护矩阵 / DDP

- **核心思想**：将 `tag` 和 `val` 设计为矩阵，`pushup` / `pushdown` 变为矩阵乘法（通常是 Min-Plus 矩阵乘法 $C[i][j]=\min_k(A[i][k]+B[k][j])$）
- **典型场景一**：多标记统一（区间加 + 区间乘 + 区间赋值 → 矩阵形式）
- **典型场景二**：区间斐波那契 / 递推问题，利用转移矩阵进行区间修改
- **典型场景三 — DDP（动态 DP）**：
  - 将 DP 转移写成矩阵形式，用线段树维护区间矩阵乘积
  - 例题 CF750E：维护与 `2017` 匹配所需最少删除数，DP → Min-Plus 矩阵 → 线段树
  - 例题 洛谷 P4314：历史最大值，设计加矩阵和赋值矩阵
- **注意事项**：矩阵乘法不满足交换律，pushdown 顺序至关重要

## 进阶专题（选讲）

### 树套树

- **思想**：用 DS 套 DS 维护多维信息，第一维值域不大时用树状数组做外层，否则用动态开点线段树
- **典型结构**：树状数组套权值线段树（BIT + Segment Tree）
  - 外层 BIT 维护 $x$ 轴，每个 BIT 节点是一棵动态开点权值线段树维护 $y$ 轴
  - 单点修改：BIT 上 $O(\log n)$ 个节点各插一次，总 $O(\log^2 n)$
  - 区间查询：BIT 前缀差 + 线段树二分
- **应用**：二维数点、动态区间第 K 小（带修改的主席树）
- **线段树套线段树 + 标记永久化**：外层线段树维护 $x$，内层维护 $y$，查询矩形最大值

> 📖 题解 `H`、`I` 有完整实现

### 线段树优化建图

- **思想**：利用线段树的区间覆盖能力，将 $O(n)$ 条边优化为 $O(\log n)$ 条
- **实现**：建两棵线段树（一棵父亲→儿子连边，一棵儿子→父亲连边），叶节点对应原图节点
  - 点→区间连边：从原始节点连向第一棵树的 $O(\log n)$ 个节点
  - 区间→点连边：从第二棵树的 $O(\log n)$ 个节点连向原始节点
- **应用**：区间连边后跑最短路（Dijkstra）

> 📖 题解 `A` 有完整代码

### 吉司机线段树（Segment Tree Beats）

- **场景**：区间取 $\min$、区间取 $\max$、历史最大值等非均等修改
- **核心技巧**：维护最大值和次大值，只在 $se < val < max$ 时修改（此时只有最大值被修改，可转化为区间加）
- **懒标记设计**：需维护四组标记（最大值加法、非最大值加法、及其历史最大值版本）
- **复杂度**：均摊 $O(m \log n)$，实际常数较大
