# 同余最短路

> 最小值做模数，每个同余类只关心最小的可凑出值，剩下等差数列直接算。

---

## 适用场景

给定 $n$ 个正整数 $a_1, a_2, \dots, a_n$（每种无限使用），问：

- 不超过 $H$ 的数中有多少个能被凑出？
- 能否凑出某个特定值 $X$？
- 第 $k$ 小的能被凑出的数是多少？

数据特征：$H$ 可到 $10^{18}$ 级别，但 $a_i$ 范围可控（$\min\{a_i\} \le 10^5$）。

---

## 核心洞察

设 $m = \min\{a_i\}$。对于任意能被凑出的数 $S$：

$$S = k \cdot m + r \quad (0 \le r < m)$$

同余类 $r$ 中的数构成公差为 $m$ 的等差数列。**每个同余类只需要知道最小的那个可凑出值**，因为后续所有值可以通过不断加 $m$ 得到。

---

## 建图跑最短路

### 状态定义

$$\text{dis}[x] = \text{在模 } m \text{ 意义下同余于 } x \text{ 的数中，能被凑出的最小值}$$

### 建图

- 节点：$0, 1, \dots, m-1$（$m$ 个同余类）
- 边：$x \xrightarrow{a_j} (x + a_j) \bmod m$，边权为 $a_j$

### 初始条件

$$\text{dis}[0] = 0,\quad \text{dis}[x] = +\infty \;(x \neq 0)$$

### 转移（Dijkstra）

从当前节点 $u$ 出发，枚举每个 $a_j$：

$$v = (u + a_j) \bmod m,\quad \text{dis}[v] = \min(\text{dis}[v],\; \text{dis}[u] + a_j)$$

### 复杂度

- 节点数：$O(m)$，其中 $m = \min\{a_i\}$
- 边数：每个节点连 $n$ 条边
- 总复杂度：$O(nm \log m)$

---

## 答案统计

对于上限 $H$：

$$\text{ans} = \sum_{x=0}^{m-1} \max\left(0,\; \left\lfloor \frac{H - \text{dis}[x]}{m} \right\rfloor + 1\right)$$

> **注意**：当 $\text{dis}[x] > H$ 时该同余类无贡献，$\max(0, \dots)$ 处理了这种情况。

区间查询 $[L, R]$：$\text{ans} = f(R) - f(L-1)$。

### 推导

设同余类 $x$ 中最小的可凑出值为 $\text{dis}[x]$，同余类中可凑出的数集合为：

$$\{\text{dis}[x],\; \text{dis}[x] + m,\; \text{dis}[x] + 2m,\; \dots\}$$

不超过 $H$ 的项数：

$$\left\lfloor \frac{H - \text{dis}[x]}{m} \right\rfloor + 1$$

由于被 $m$ 除后向下取整再加一，等价于该等差数列在 $[0, H]$ 中的项数。

---

## 正确性证明（一句话）

任意可凑出数 $S$ 可写为 $S = \text{dis}[S \bmod m] + t \cdot m$（$t \ge 0$），因为加 $m$ 不改变同余类，且 $\text{dis}[S \bmod m]$ 是该类的最小值——若存在更小的路径则 Dijkstra 已更新。

## 模板代码

```cpp
// 同余最短路 — 模板
// 适用：给定 n 个数，每种无限使用，统计不超过 H 的可凑出数量
// 复杂度：O(n * m * log m)，m = min{a_i}

#include <bits/stdc++.h>
using namespace std;
using ll = long long;

const ll INF = 1e18;

ll solve(ll H, vector<ll>& a) {
    int n = a.size();
    ll m = *min_element(a.begin(), a.end());

    // 含 1 的直接特判：所有数都能凑出
    if (m == 1) return H + 1;

    vector<ll> dis(m, INF);
    dis[0] = 0;
    priority_queue<pair<ll, int>, vector<pair<ll, int>>, greater<>> pq;
    pq.push({0, 0});

    while (!pq.empty()) {
        auto [d, u] = pq.top(); pq.pop();
        if (d != dis[u]) continue;
        for (ll x : a) {
            int v = (u + x) % m;
            if (dis[v] > dis[u] + x) {
                dis[v] = dis[u] + x;
                pq.push({dis[v], v});
            }
        }
    }

    ll ans = 0;
    for (int i = 0; i < m; i++) {
        if (dis[i] <= H)
            ans += (H - dis[i]) / m + 1;
    }
    return ans;
}
```

### 常见坑点

- **$m=1$ 特判**：所有数都能凑出，直接返回 $H+1$，否则 dis 数组大小 0 会 RE。
- **INF 取值**：需大于 $H + \max\{a_i\}$，推荐 `1e18`。
- **统计时用 `(H - dis[i]) / m`**：必须用 `H - dis[i]` 而非 `H`，因为 dis[i] 本身已是当前同余类的第一个可凑出值。

## 识别指南

看到以下关键词时考虑同余最短路：

- **"无限使用" / "每种任意次"**：背后是模 $m$ 的等差数列结构
- **"凑数" / "可达" / "能被表示"**：典型的状态建模问题
- **$H$ 极大（$10^{18}$）但 $a_i$ 较小**：排除 DP / 背包后，大概率是同余类技巧

### 常见伪装

- "跳楼机有 $x,y,z$ 三个按钮，求能到达的楼层" → 本质上就是给定三个数，求可凑出的值个数。
- "用若干面值的硬币支付，求不能支付的金额数量" → 类似结构，$\min$ 做模数。

## 扩展

### 第 K 小可凑出值

先用同余最短路求出 $\text{dis}$，然后二分答案 $X$，用统计公式判断不超过 $X$ 的个数是否 $\ge k$。

### 多询问能否凑出 $X$

预处理好 $\text{dis}$ 后，单次查询 $O(1)$：检查 $X \ge \text{dis}[X \bmod m]$ 即可。
